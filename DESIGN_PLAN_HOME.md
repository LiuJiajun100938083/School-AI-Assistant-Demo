# 系統首頁 UI 重設計方案

## 一、現狀診斷

### 核心問題

| 問題 | 具體表現 |
|------|----------|
| **信息層級為零** | 所有應用卡片同尺寸、同樣式、同權重，形成均質網格 |
| **功能展示頁而非產品首頁** | 標題「AI 學習應用助手 → 選擇下方應用開始使用」是目錄思維，不是任務引導 |
| **無用戶上下文** | 不顯示今日學習狀態、待辦任務、最近活動等個人化信息 |
| **視覺節奏扁平** | `grid-template-columns: repeat(auto-fill, minmax(150px, 1fr))` → 每張卡片完全等大 |
| **emoji 圖標** | 💬🧠🎯📢📊 等 emoji 作為圖標，缺乏統一的視覺語言 |
| **hover 效果廉價** | `translateY(-4px) scale(1.02)` + 綠色背景漂移，像模板站 |
| **無行動引導** | 用戶登入後看到 9-13 個等權按鈕，沒有明確「你現在該做什麼」 |

**參考產品**：Stripe Dashboard / Linear / Notion Home / Things 3
**核心原則**：引導行動、層次分明、克制專業

## 三、信息優先級排序

### 用戶角色：學生（主要用戶）

| 優先級 | 信息/功能 | 理由 |
|--------|-----------|------|
| P0 | 問候 + 今日摘要（待複習錯題、未完成任務、學習連續天數） | 用戶需要知道「我今天該做什麼」 |
| P1 | AI 學習對話（主入口） | 最高頻功能，1 對 1 AI 輔導 |
| P1 | AI 錯題本 | 核心學習閉環：拍題→分析→練習 |
| P1 | AI 學習中心 | 多維度互動學習 |
| P2 | 學習任務 | 老師指定的任務 |
| P2 | 學習分析 | 查看學習數據 |
| P2 | 遊戲中心 | 寓教於樂 |
| P2 | 討論區 | 社區功能 |
| P3 | 課堂教學、考勤、通告等 | 教師/管理員功能，學生不常用 |

## 五、組件尺寸規劃

### 5.1 卡片尺寸系統

| 級別 | 用途 | 高度 | 寬度 | 視覺權重 |
|------|------|------|------|----------|
| **Hero Card** | 主任務區前 2 個（AI 對話、錯題本） | 160px | 各佔 1/2（gap 16px） | ★★★★★ |
| **Feature Card** | 主任務區第 3 個（AI 學習中心） | 120px | 全寬 | ★★★★ |
| **Quick Card** | 快捷操作區（學習任務、分析、遊戲、討論） | 80px | 各佔 1/4（gap 12px） | ★★★ |
| **List Item** | 更多功能區 | 48px | 全寬 | ★★ |

### 5.2 視覺節奏

```
歡迎區:   ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  （統計指標行）
主任務區:  ████████████  ████████████       （160px 雙列）
          ██████████████████████████████    （120px 全寬）
快捷操作:  ████  ████  ████  ████            （80px 四列）
更多功能:  ─────────────────────────────      （48px 列表行）
```

大→大→中→小→極小，明確的遞減節奏。

## 六、UI 風格定義

### 6.1 卡片樣式

| 級別 | 背景 | 邊框 | 圓角 | 陰影 | 內距 |
|------|------|------|------|------|------|
| **Hero Card** | `#FFFFFF` | `1px solid rgba(0,0,0,0.06)` | 20px | `shadow-card` | 28px |
| **Feature Card** | `var(--brand-lighter)` #F0F9F3 | `1px solid rgba(0,102,51,0.08)` | 20px | `shadow-sm` | 24px |
| **Quick Card** | `#FFFFFF` | `1px solid rgba(0,0,0,0.06)` | 16px | `shadow-xs` | 20px |
| **List Item** | transparent | bottom `1px solid rgba(0,0,0,0.04)` | 0 | none | 16px 0 |

### 6.2 Hero Card 內部結構

```
┌─────────────────────────────────────┐
│  [SVG 圖標 40×40 · 綠底圓角容器]     │  ← 左上角
│                                     │
│  AI 學習對話                         │  ← 標題 18px / 600
│  一對一AI輔導，隨時解答學習疑問        │  ← 描述 14px / 次要色
│                                     │
│  開始對話  →                         │  ← 底部 CTA 文字鏈
└─────────────────────────────────────┘
```

### 6.3 Quick Card 內部結構

```
┌──────────────────┐
│  [SVG 24×24]      │
│  學習任務          │  ← 14px / 600

```

### 6.4 List Item 結構

```

─────────────────────────────────────────────────
```

### 6.5 圖標系統

統一 SVG 線性圖標（Lucide 風格，24×24，2px stroke，round join）：

| 應用 | 圖標 |
|------|------|
| AI 學習對話 | `message-circle`（對話泡泡） |
| AI 錯題本 | `book-open`（打開的書） |
| AI 學習中心 | `compass`（指南針/探索） |
| 學習任務 | `check-square`（勾選方框） |
| 學習分析 | `bar-chart-2`（柱狀圖） |
| 遊戲中心 | `target`（靶心） |
| 討論區 | `message-square`（方形對話框） |
| 課堂教學 | `monitor`（螢幕） |
| 考勤點名 | `clipboard-list`（清單） |
| 通告生成 | `file-text`（文件） |
| 錯題分析(教師) | `pie-chart`（餅圖） |
| 任務管理 | `settings`（齒輪） |
| 上傳遊戲 | `upload`（上傳） |
| 管理後台 | `sliders`（滑桿控制） |

Hero Card 圖標容器：40×40px，品牌綠淡底（`#E8F5EC`），圓角 12px，圖標 24×24 白色或綠色。
Quick Card 圖標：24×24，品牌綠色，無容器。
List Item 圖標：20×20，`#6E6E73` 灰色。

### 6.6 歡迎區樣式

```
早安，劉同學                        ← 24px / 700 / #1D1D1F
今天是星期一，準備好學習了嗎？         ← 14px / 400 / #6E6E73

┌──────────┐ ┌──────────┐ ┌──────────┐
│   3       │ │   2      │ │   5      │     ← 數字 28px / 700 / 品牌綠
│  待複習    │ │  待完成   │ │  連續天數  │     ← 標籤 12px / 500 / #6E6E73
└──────────┘ └──────────┘ └──────────┘

指標卡片：白底、1px 邊框、12px 圓角、寬度各佔 1/3
```

**API 數據來源**：
- 待複習錯題數 → `GET /api/mistakes/dashboard`
- 待完成任務數 → `GET /api/learning-tasks`（篩選未完成）
- 連續學習天數 → 前端根據簽到記錄計算 或 後端提供

## 七、配色策略

### 使用規則

| 用途 | 顏色 | 面積 |
|------|------|------|
| 頁面底色 | `#F5F5F7` | 最大面積 |
| 卡片底色 | `#FFFFFF` | 大面積 |
| 主文字 | `#1D1D1F` | 正常 |
| 次要文字 | `#6E6E73` | 正常 |
| 品牌綠底色 | `#F0F9F3` | 僅 Feature Card 底色 |
| 品牌綠 | `#006633` | 僅用於：Hero 圖標容器、CTA 文字、統計數字、active 狀態 |
| 語義色 | 紅/橙/綠 | 僅用於統計指標的小圓點或 badge |

**80% 中性色 + 品牌綠點綴** — 綠色只出現在需要吸引注意力的地方。

## 八、交互規範

### 允許的動效

| 場景 | 動效 | 參數 |
|------|------|------|
| 卡片 hover | 背景色微變 + 陰影加深 | 150ms ease-out |
| CTA hover | 文字顏色加深 + `→` 右移 2px | 150ms ease-out |
| 更多功能展開 | height + opacity 漸變 | 200ms ease |
| 統計數字 | 進入視口時數字漸現 | 300ms ease-out |

### 禁止的動效

- `translateY` 上浮效果
- `scale` 縮放效果
- 顏色閃爍
- 卡片邊框顏色漸變到品牌色

### 交互邏輯

1. **Hero Card** 點擊 → 跳轉對應功能頁
2. **Quick Card** 點擊 → 跳轉對應功能頁
3. **List Item** 點擊 → 跳轉對應功能頁
4. **更多功能** 默認收起，點擊展開（教師/管理員功能，學生一般不需要）
5. **統計指標卡** 點擊 → 跳轉到對應詳情（如點擊「3 道待複習」→ 跳轉錯題本）

## 十、實現改造清單

### HTML 結構變化

1. 歡迎區：新增 `.home-welcome` 區塊（問候語 + 統計指標行）
2. 主任務區：`.home-hero-grid`（2 個 Hero Card + 1 個 Feature Card）
3. 快捷操作區：`.home-quick-grid`（4 個 Quick Card）
4. 更多功能區：`.home-more-section`（可折疊列表）
5. 刪除 `.home-title-section`（「AI 學習應用助手」+ 「選擇下方應用」）
6. 刪除 `.home-apps-grid`（均質卡片網格）

### CSS 變化

1. 新增 Hero / Feature / Quick / ListItem 四級卡片樣式
2. 刪除 `.home-app-card` 的 translateY/scale hover
3. 新增歡迎區、統計指標行樣式
4. 新增折疊展開動效
5. 重寫響應式規則

### JS 變化

1. `renderHomeApps()` 重構：根據 app.category + 手動優先級分配到 4 個層級
2. 新增 SVG 圖標映射表（app.id → SVG path）
3. 新增歡迎語邏輯（根據時間段切換早安/午安/晚安）
4. 新增統計數據載入（錯題數、任務數、連續天數）
5. 新增「更多功能」折疊/展開邏輯
6. 歡迎區統計指標可點擊跳轉

## 十一、效果預期

| 維度 | 現狀 | 改造後 |
|------|------|--------|
| 首屏體驗 | 「選一個應用」 | 「早安，你有 3 道錯題待複習」 |
| 視覺層級 | 扁平均質網格 | Hero(160px)→Feature(120px)→Quick(80px)→List(48px) |
| 行動引導 | 無（9 個等權按鈕） | 明確的 2 個主入口 + 統計跳轉 |
| 品牌感 | 模板頁 | 像一個真正的教育產品 |
| 功能發現 | 所有功能同等暴露 | 高頻功能突出 + 低頻功能收納 |
| 圖標一致性 | emoji 混搭 | 統一 SVG 線性圖標 |

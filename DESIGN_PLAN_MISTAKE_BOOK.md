# AI 智能錯題本 — UI 重設計方案

## 一、現狀診斷

### 核心問題

| 問題 | 具體表現 |
|------|----------|
| **視覺層級扁平** | 統計卡片、功能入口、錯題列表三個區塊視覺權重幾乎一樣，沒有主次 |
| **卡片同質化** | 所有卡片同尺寸、同圓角、同陰影，像"模板拼裝" |
| **emoji 代替圖標** | 底部 tab 和功能入口全用 emoji（📋🔄✏️📊📷✍️🎯🧠），缺乏統一圖標語言 |
| **留白不足** | padding 偏小（16-20px），模塊之間間距均勻但沒有"呼吸感" |
| **配色過散** | 科目色（紅/藍/綠）+ 狀態色（藍/橙/綠/灰）+ 品牌色，顏色太多且無主次 |
| **底部 tab 廉價感** | emoji + 文字的 tab bar 像小程序模板 |
| **分析頁堆砌** | 雷達圖、薄弱點、趨勢、AI 建議、知識樹全平鋪，信息過載 |

---

## 二、設計方向

**參考產品**：Apple Education / Notion / Linear / Things 3
**核心原則**：克制、層次、真實、耐看

## 三、新版信息層級規劃

### 首頁（Home Tab）— 重新分區

```
┌──────────────────────────────────────────────┐
│  Header（品牌+科目切換）                       │
├──────────────────────────────────────────────┤
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │  Hero Card（焦點卡）                    │  │
│  │  "你有 X 道錯題待複習"                   │  │
│  │  [立即複習] 按鈕                        │  │
│  │  小字：平均掌握度 67%  ·  連續複習 5 天   │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  ┌──────────┐ ┌──────────┐                  │
│  │ 拍照上傳  │ │ 手動添加  │  ← 兩個主操作   │
│  └──────────┘ └──────────┘                  │
│                                              │
│  最近錯題 ─────────────── 查看全部 >          │
│  ┌────────────────────────────────────────┐  │
│  │  錯題卡片 1                             │  │
│  ├────────────────────────────────────────┤  │
│  │  錯題卡片 2                             │  │
│  ├────────────────────────────────────────┤  │
│  │  錯題卡片 3                             │  │
│  └────────────────────────────────────────┘  │
│                                              │
└──────────────────────────────────────────────┘
```

**關鍵變化**：
1. **Hero Card 取代統計行**：3 個同等權重的小統計卡→1 個大焦點卡，突出"最該做的事"
2. **操作入口縮減**：4 個功能按鈕→2 個（拍照、手動），"AI 練習"和"今日複習"歸入 tab
3. **錯題列表用列表式**而非獨立卡片，更緊湊、可掃描性更好

### 複習頁（Review Tab）— 保持簡潔

沿用卡片翻轉交互，但視覺升級：
- 卡片更大、字體更大、中心對齊
- 底部三個按鈕改為語義色條（紅/黃/綠）而非獨立按鈕
- 進度指示改為頂部薄進度條

### 練習頁（Practice Tab）— 沉浸式

- 去除冗余裝飾
- 題目區域全寬、大字
- 答題區域有明確 focus 狀態
- 提交後結果頁：分數用大數字展示，每題折疊式展開查看

### 分析頁（Analysis Tab）— 分層折疊

**核心問題**：現在 5 個模塊平鋪，信息過載。

**新方案**：分兩層
- **第一層**（默認可見）：概覽摘要 + 最需攻克的知識點（最多 3 個）
- **第二層**（點擊展開）：雷達圖、趨勢圖、知識樹、AI 完整建議

```
┌────────────────────────────────────────┐
│  學習概覽（1-2 句 AI 摘要）             │
│  ──────────────────────                │
│  最需攻克                              │
│  ┌─────────────────────────────────┐   │
│  │  知識點 A    32%   ↓            │   │
│  │  知識點 B    45%   →            │   │
│  │  知識點 C    51%   ↑            │   │
│  └─────────────────────────────────┘   │
│                                        │
│  [掌握度總覽 ▸]  ← 展開雷達圖          │
│  [趨勢變化 ▸]    ← 展開折線圖          │
│  [知識點全覽 ▸]  ← 展開樹狀圖          │
│  [完整 AI 建議 ▸]                      │
└────────────────────────────────────────┘
```

## 四、組件風格定義

### 4.1 卡片系統（3 級）

| 級別 | 用途 | 樣式 |
|------|------|------|
| **Hero Card** | 首頁焦點卡、分數展示 | 大圓角（20px）、品牌色淡底（`#F0F9F3`）、無邊框、padding 32px |
| **Content Card** | 錯題詳情區塊、圖表容器 | 中圓角（16px）、白底、1px border `rgba(0,0,0,0.06)`、`shadow: 0 1px 3px rgba(0,0,0,0.04)` |
| **List Item** | 錯題列表項 | 無圓角（在容器內用分隔線）、hover 背景色 `#FAFAFA`、左側 3px 狀態條 |

### 4.2 按鈕系統

| 類型 | 樣式 | 用途 |
|------|------|------|
| **Primary** | 品牌綠底白字、圓角 12px、shadow | "立即複習""開始練習""確認並分析" |
| **Secondary** | 白底、1px border、品牌綠字 | "拍照上傳""手動添加" |
| **Ghost** | 無背景無邊框、品牌綠字 | "查看全部""展開" |
| **Danger** | 紅色文字、無背景 | "刪除" |

### 4.3 圖標

**統一使用 SVG inline icon**，取代所有 emoji。

用 Lucide Icons 風格（24x24、2px stroke、round join）：
- 錯題 tab → `book-open`
- 複習 tab → `repeat`
- 練習 tab → `pen-tool`
- 分析 tab → `bar-chart-2`
- 拍照上傳 → `camera`
- 手動添加 → `edit-3`
- 返回 → `chevron-left`
- 展開 → `chevron-right`

### 4.4 排版

| 元素 | 規格 |
|------|------|
| 頁面標題 | 22px / 700 / -0.02em |
| 區塊標題 | 15px / 600 / -0.01em / `#1D1D1F` |
| 正文 | 15px / 400 / 1.6 / `#1D1D1F` |
| 輔助文字 | 13px / 500 / `#6E6E73` |
| 標籤/Badge | 12px / 600 / `#A1A1A6` |
| 大數字（Hero） | 40px / 700 / -0.03em |

## 五、配色策略

### 主色盤

```
品牌色（僅用於強調）
  Primary:    #006633    ← CTA 按鈕、active 狀態、品牌標識
  Light:      #E8F5EC    ← Hero 卡背景、hover tint
  Lighter:    #F0F9F3    ← 極淡底色

中性色（80% 的頁面）
  Text:       #1D1D1F    ← 主文字
  Secondary:  #6E6E73    ← 次要文字
  Tertiary:   #A1A1A6    ← 輔助/禁用
  Background: #F5F5F7    ← 頁面底色
  Card:       #FFFFFF    ← 卡片底色
  Border:     rgba(0,0,0,0.06)  ← 分隔線
```

### 語義色（僅用於狀態指示，小面積）

```
  Success:  #34C759    ← 已掌握、正確
  Warning:  #FF9500    ← 練習中、部分正確
  Danger:   #FF3B30    ← 薄弱、錯誤
  Info:     #007AFF    ← 已分析、鏈接
```

### 配色規則

1. **品牌綠**只用於：主按鈕、active tab、Hero 卡淡底、進度條高值段
2. **語義色**只用於：status badge、進度條、趨勢箭頭、結果指示
3. **科目色不用彩色背景**：改用文字顏色區分 + 統一的灰底 badge
4. **頁面 80% 以上面積**為白/灰/黑中性色

## 六、布局結構

### 頁面框架

```
┌──────────────────────────────────────┐
│  Header（sticky）                     │  48px
│  半透明毛玻璃 + 品牌名                 │
│  科目切換（嵌入 header 右側）           │
├──────────────────────────────────────┤
│                                      │
│  Content Area                        │  flex-grow
│  max-width: 680px (mobile-first)     │
│  padding: 0 20px                     │
│                                      │
├──────────────────────────────────────┤
│  Tab Bar（fixed bottom）             │  49px + safe area
│  SVG icons + label                   │
└──────────────────────────────────────┘
```

### 響應式斷點

| 寬度 | Content max-width | 特殊調整 |
|------|-------------------|----------|
| < 768px | 100% | 移動端默認 |
| 768-1024px | 680px | 居中 |
| 1024-1280px | 760px | |
| > 1280px | 840px | |

### 間距系統

- 區塊間距：32px（大分組之間）
- 卡片間距：16px
- 卡片內 padding：20-24px
- 列表項間距：0（用分隔線）

## 七、交互規範

### 允許的動效

| 場景 | 動效 | 參數 |
|------|------|------|
| Tab 切換 | 內容 fade in | 150ms ease |
| 卡片 hover | 背景色漸變 | 150ms ease |
| 面板彈出 | 底部滑入 | 300ms cubic-bezier(0.16,1,0.3,1) |
| 進度條 | 寬度漸變 | 400ms ease |
| 折疊展開 | height 動畫 | 200ms ease |
| Toast | 頂部滑入+淡出 | 300ms |

### 禁止的動效

- 卡片 hover translateY（"漂浮感"不必要）
- active scale（按下縮放感覺像小程序）
- 3D 翻轉（複習卡不用翻轉，改為"點擊展開答案"）

## 八、具體改造清單

### HTML 結構變化

1. 科目切換從獨立條移到 header 右側（省空間）
2. Hero card 取代 stats grid
3. 操作按鈕從 4 宮格→2 個並排 secondary 按鈕
4. 錯題列表從獨立卡片→連續列表（list item + divider）
5. 底部 tab bar 的 emoji 換 SVG

### CSS 變化

1. 刪除所有 hover translateY 動效
2. 刪除所有 active scale 動效
3. 卡片陰影降低（更微妙）
4. 圓角分級（Hero 20px、卡片 16px、按鈕 12px、badge 6px）
5. 留白加大（section gap 32px）
6. 分析頁改為折疊式

### JS 變化

1. `renderHome`：重構為 Hero + 操作 + 列表 三段式
2. `renderAnalysis`：拆分為摘要層+折疊層
3. `_renderReviewCard`：去掉 3D 翻轉，改為展開式
4. 所有 emoji icon 替換為 SVG helper 函數
5. 科目切換事件綁定位置調整

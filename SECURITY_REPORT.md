# AI 学习伙伴 — 网络安全保障报告

> 版本: 2.0.0 | 日期: 2026 年 3 月 31 日 | 部署环境: Mac Studio (Docker)

---

## 一、系统架构安全

### 1.1 容器化隔离

应用运行在 Docker 容器内，与宿主机操作系统隔离。

| 措施 | 说明 |
|------|------|
| 非 root 运行 | 容器内以专用 `appuser` 身份运行，无系统管理权限 |
| 文件系统隔离 | 应用只能访问明确挂载的目录，无法读写宿主机其他文件 |
| 内存限制 | 容器设有内存上限，防止资源耗尽影响其他服务 |
| 自动恢复 | 容器异常退出后自动重启，保障服务可用性 |
| 信号转发 | 启用 `init` 进程，确保子进程正确终止，防止僵尸进程 |

### 1.2 网络隔离

| 端口 | 服务 | 暴露范围 | 保护措施 |
|------|------|----------|----------|
| 8002 | Web 应用 | 仅本机 + Ngrok 隧道 | HTTPS 加密 (TLS 1.3) |
| 3306 | MySQL 数据库 | 仅本机 | macOS pf 防火墙阻止外部访问 |
| 11434 | Ollama AI | 仅本机 | 绑定 localhost |

## 二、数据安全

### 2.1 数据库安全

| 措施 | 说明 |
|------|------|
| 专用账户 | 使用 `pkms_app` 专用数据库账户，非 root，遵循最小权限原则 |
| 防火墙保护 | macOS pf 规则阻止所有外部网络访问 MySQL 端口 |
| 参数化查询 | 所有 SQL 使用参数化查询 (`%s` 占位符)，从根本上防止 SQL 注入攻击 |
| 连接池管理 | 使用 DBUtils 连接池，自动回收超时连接，防止连接泄露 |
| 事务管理 | 关键操作使用事务，失败自动回滚，保障数据一致性 |

### 2.2 数据备份

| 措施 | 说明 |
|------|------|
| 每日自动备份 | 每天凌晨 3:00 自动执行 `mysqldump`，压缩存储 |
| 保留策略 | 保留最近 7 天的备份文件，自动清理过期备份 |
| 备份日志 | 每次备份记录执行结果和文件大小，便于审计 |

### 2.3 密钥管理

| 措施 | 说明 |
|------|------|
| 环境变量隔离 | 所有敏感配置（数据库密码、JWT 密钥、API 密钥）通过环境变量注入，不存入代码仓库 |
| .gitignore 保护 | `.env`、密钥文件被 `.gitignore` 排除，杜绝意外提交 |
| .dockerignore 保护 | Docker 镜像构建时排除所有密钥文件，镜像内无任何敏感信息 |
| 文件权限加固 | 启动时自动将密钥文件权限设为 `0600`（仅所有者可读写） |

## 三、身份认证与授权

### 3.1 密码安全

| 措施 | 说明 |
|------|------|
| bcrypt 加密 | 密码使用 bcrypt 算法（12 轮 salt）单向加密存储，即使数据库泄露也无法还原明文 |
| 强密码策略 | 要求 8-128 字符，必须包含大写字母、小写字母、数字、特殊字符 |
| 弱密码检测 | 内置常见弱密码黑名单（123456、password、admin 等），自动拒绝 |
| 常量时间比较 | 密码验证使用常量时间比较，防止计时攻击 |

### 3.2 JWT 令牌管理

| 措施 | 说明 |
|------|------|
| 短期访问令牌 | Access Token 有效期 24 小时，过期需重新认证 |
| 刷新令牌轮换 | Refresh Token 使用后立即作废，签发新令牌，防止重放攻击 |
| 令牌撤销 | 支持单令牌撤销和家族级撤销（一个被盗，全部作废） |
| 黑名单持久化 | 撤销的令牌记录到数据库，服务重启后仍然有效 |
| 内存缓存加速 | 令牌验证使用内存缓存，兼顾安全性和性能 |

### 3.3 登录保护

| 措施 | 说明 |
|------|------|
| 三级封锁机制 | 同一 IP+用户名：5 次失败封锁；同一用户名：10 次封锁；同一 IP：200 次封锁 |
| 智能学校 IP 处理 | 识别学校共享 IP 环境，避免一个学生的错误登录影响其他学生 |
| 时间窗口滑动 | 10 分钟滑动窗口，超时自动解封 |
| IP 白名单 | 管理员 IP 可设置白名单豁免 |
| 审计日志 | 每次登录成功/失败都记录 IP、时间、用户名，可追溯 |

## 四、Web 应用安全

### 4.1 HTTP 安全头

| 安全头 | 配置 | 防护目标 |
|--------|------|----------|
| Content-Security-Policy | 严格白名单控制脚本、样式、图片来源 | XSS 攻击 |
| Strict-Transport-Security | max-age=31536000; includeSubDomains | 强制 HTTPS |
| X-Content-Type-Options | nosniff | MIME 类型嗅探 |
| X-Frame-Options | DENY (默认) / SAMEORIGIN (游戏页面) | 点击劫持 |
| Referrer-Policy | strict-origin-when-cross-origin | 信息泄露 |
| Permissions-Policy | 禁用摄像头、麦克风、地理位置 | 隐私保护 |

### 4.2 XSS 防护

| 措施 | 说明 |
|------|------|
| HTML 净化 | 使用 bleach 库净化用户输入，只允许安全的 HTML 标签和属性 |
| 脚本标签过滤 | 自动移除 `<script>`、`<iframe>`、`<style>` 等危险标签 |
| 事件处理器过滤 | 自动移除 `onclick`、`onload` 等 JavaScript 事件属性 |
| JavaScript URL 过滤 | 自动移除 `javascript:` 链接 |
| 文本转义 | 使用 `html.escape()` 对纯文本输出进行转义 |

### 4.3 CSRF 防护

| 措施 | 说明 |
|------|------|
| JWT Bearer 认证 | 使用 HTTP Authorization 头传递令牌，浏览器不会自动发送，天然抵抗 CSRF |
| CORS 策略 | 严格限制允许的跨域来源，拒绝未授权域名的请求 |

### 4.4 API 限流

| 路径模式 | 限制 | 目的 |
|----------|------|------|
| 登录/注册 | 10 次/分钟 | 防暴力破解 |
| AI 游戏 | 5 次/分钟 | 防 AI 资源滥用 |
| 文件上传 | 10 次/分钟 | 防存储攻击 |
| 管理接口 | 60 次/分钟 | 防管理员账户滥用 |
| 通用 API | 60 次/分钟 | 防 DDoS |

## 五、内容安全

### 5.1 AI 内容审核

| 措施 | 说明 |
|------|------|
| 实时审核 | 所有用户发布的内容经过 AI 模型 (qwen3.5:4b) 实时审核 |
| 审核类别 | 涉黄、暴力、毒品、恐怖主义、儿童安全、仇恨言论 |
| 失败关闭 | 审核系统故障时默认拒绝内容，宁可误拦不可放行 |
| 熔断保护 | 审核服务异常时自动熔断，防止级联故障 |
| 无内容日志 | 审核日志只记录元数据（决定、类别、延迟），不记录原文，保护隐私 |

### 5.2 文件上传安全

| 措施 | 说明 |
|------|------|
| 文件类型限制 | 仅允许 `.html`、`.htm` 格式上传 |
| 文件大小限制 | 单文件最大 5MB |
| 编码验证 | 验证文件为合法 UTF-8 编码 |

## 六、审计与监控

### 6.1 安全审计日志

系统记录以下安全事件，存入 MySQL `security_audit_log` 表：

| 事件类型 | 记录内容 |
|----------|----------|
| LOGIN_SUCCESS / LOGIN_FAILURE | 用户名、IP、时间 |
| LOGOUT / LOGOUT_ALL | 用户名、IP |
| TOKEN_REFRESH / TOKEN_REFRESH_REPLAY | 令牌使用情况 |
| PASSWORD_CHANGED / PASSWORD_RESET | 密码变更操作 |
| ACCOUNT_LOCKED / ACCOUNT_UNLOCKED | 账户封锁/解封 |
| RATE_LIMIT_TRIGGERED | 触发限流的 IP 和路径 |

### 6.2 应用监控

| 措施 | 说明 |
|------|------|
| 健康检查 | Docker 每 30 秒检查应用健康状态 (`/health` 端点) |
| 资源监控 | `docker stats` 实时查看 CPU/内存使用 |
| 日志管理 | 所有日志输出到文件，可通过 `docker compose logs` 查看 |

## 七、传输安全

| 措施 | 说明 |
|------|------|
| HTTPS 加密 | 所有外部访问通过 Ngrok 隧道，自动提供 TLS 1.3 加密 |
| HSTS 强制 | 浏览器被指示始终使用 HTTPS 连接，有效期 1 年 |
| WebSocket 加密 | WebSocket 连接同样通过 WSS (加密) 通道 |

## 八、隐私保护

| 措施 | 说明 |
|------|------|
| 密码不可逆 | bcrypt 单向加密，即使系统管理员也无法查看用户密码 |
| 审核无原文 | 内容审核日志不保存用户原始内容 |
| 最小权限 | 数据库账户仅拥有必要权限，无法导出或修改其他数据库 |
| 摄像头/麦克风禁用 | 浏览器 Permissions-Policy 禁止网页访问摄像头和麦克风 |
| 缓存控制 | 静态资源设置 no-cache，防止敏感页面被浏览器缓存 |

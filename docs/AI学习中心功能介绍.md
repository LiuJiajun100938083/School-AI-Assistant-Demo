# AI 学习中心 — 功能介绍

## 一、产品概述

**AI 学习中心**是一个集教学资料管理、知识图谱可视化、学习路径规划和 AI 智能问答于一体的综合性教育平台模块。该模块旨在帮助学生高效获取学习资源，并通过 AI 助教实现"读到哪、问到哪"的沉浸式学习体验。

---

## 二、核心功能模块

### 2.1 教学资料库

为学生提供电子教科书式的内容浏览体验。

**主要能力：**
- 左侧目录树导航，支持多级分类，可折叠/展开
- 支持多种内容类型：PDF 文档、本地视频、外链视频（YouTube / Bilibili）、图片、富文本文章
- PDF 内嵌阅读器，视频在线播放，无需下载
- 自动记录浏览量，帮助教师了解资源热度

> 📸 *【截图位置：教学资料浏览界面 — 左侧目录 + 右侧 PDF 阅读器】*

---

### 2.2 知识地图

通过可视化知识图谱展示知识点之间的关联关系，帮助学生建立完整的知识体系。

**主要能力：**
- 基于 D3.js 的力导向图，节点可自由拖拽
- 每个知识点可自定义颜色、图标，视觉直观
- 点击节点展开详情面板，查看描述和关联的学习资料
- 节点之间的连线表达知识依赖关系（如"先修""包含"等）
- 支持缩放、重置、全屏操作

> 📸 *【截图位置：知识地图界面 — 节点图谱 + 右侧详情面板】*

---

### 2.3 学习路径

为学生提供结构化、有序的学习方案，降低自主学习的迷茫感。

**主要能力：**
- 教师可创建学习路径，设定难度等级（入门 / 进阶 / 高级）和预估学时
- 路径由多个有序步骤组成，每个步骤关联具体的学习资料或知识点
- 学生可按难度筛选适合自己的学习路径
- 时间轴式视图，学习进度一目了然

> 📸 *【截图位置：学习路径列表 + 路径详情时间轴视图】*

---

### 2.4 资源库

按分类组织的资源列表，方便学生按需查找。

**主要能力：**
- 分类树形筛选（支持多级分类）
- 多种排序方式：最新发布、最受欢迎、按名称排序
- 关键字搜索，快速定位资源
- 支持标签筛选

### 2.5 AI 助教（核心亮点）

基于 RAG（检索增强生成）技术的智能问答系统，是本模块最核心的差异化功能。

#### 2.5.1 内容感知问答

AI 助教能够"读懂"学生当前正在阅读的资料，实现精准回答。

**工作原理：**
1. 教师上传 PDF / Word / PPT 等文档时，系统自动提取全文内容
2. 文本被分割成小段（约 1000 字/段），并通过向量模型转化为语义向量，存入向量数据库
3. 学生提问时，系统先从当前文档中检索最相关的内容片段
4. 将相关片段作为上下文，结合大语言模型生成回答

**效果：** AI 的回答不是泛泛而谈，而是紧密围绕学生正在学习的具体教材内容。

> 📸 *【截图位置：阅读 PDF 时打开 AI 助教提问的界面】*

#### 2.5.2 PDF 页码精准引用

AI 回答中会标注信息来源的具体页码（如"【第5页】"），学生点击即可跳转到 PDF 对应页面，实现**"问到哪、翻到哪"**。

**技术实现：**
- 索引时按页提取文本，记录每个文本片段对应的页码范围
- AI 回答时在系统提示中引导模型标注页码
- 前端自动识别页码标注，转化为可点击链接
- 点击后 PDF 阅读器无闪烁跳转到目标页

> 📸 *【截图位置：AI 回答中包含【第X页】蓝色链接，点击跳转效果】*

#### 2.5.3 流式输出

AI 回答采用 SSE（Server-Sent Events）流式输出，逐字显示，无需等待完整回答生成，体验流畅自然。

#### 2.5.4 浮动窗口交互

AI 助教以浮动窗口形式呈现：
- 可自由拖拽位置，不遮挡阅读内容
- 支持小窗 / 大窗切换
- 可通过右下角手柄自由调整大小
- 不影响其他模块的正常使用

> 📸 *【截图位置：AI 助教浮动窗口 — 拖拽/缩放状态】*

## 三、管理后台功能

系统为教师和管理员提供完整的内容管理能力，前端通过"阅读 / 编辑 / 管理"三种模式切换。

### 3.1 分类管理
- 创建、编辑、删除分类
- 支持多级父子分类结构
- 自定义分类图标

### 3.2 内容管理
- 支持拖拽上传文件（PDF、Word、PPT、视频等）
- 支持外链视频（自动识别 YouTube / Bilibili 平台）
- 内容标签管理
- 编辑模式下可上下拖动调整内容排序
- 发布 / 下架控制

### 3.3 知识点管理
- 创建知识点节点（自定义标题、颜色、图标）
- 建立知识点之间的关联关系（边）
- 拖拽调整节点位置
- 关联学习资料到知识点

### 3.4 学习路径管理
- 创建学习路径，设定难度和预估学时
- 按顺序添加学习步骤（关联已有资料或知识点）
- 草稿 / 发布状态管理

## 四、技术架构概览

```
┌──────────────────────────────────────────────────────┐
│                    前端 (HTML/JS/CSS)                  │
│  教学资料 │ 知识地图 │ 学习路径 │ 资源库 │ AI助教     │
└──────────────────────┬───────────────────────────────┘
                       │ HTTP / SSE
┌──────────────────────┴───────────────────────────────┐
│                  FastAPI 后端服务                      │
│         路由层 → 业务逻辑层 → 数据访问层              │
└──────┬───────────────────────────────┬───────────────┘
       │                               │
┌──────┴──────┐               ┌────────┴────────┐
│   MySQL     │               │  RAG 系统        │
│  关系数据库  │               │  ChromaDB 向量库  │
│  内容/分类   │               │  文本提取 → 分块   │
│  路径/知识点 │               │  向量索引 → 检索   │
└─────────────┘               └────────┬────────┘
                                       │
                              ┌────────┴────────┐
                              │   大语言模型      │
                              │  (Ollama 本地部署) │
                              └─────────────────┘
```

**关键技术选型：**

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生 HTML/JS/CSS | 轻量、无框架依赖 |
| 知识图谱 | D3.js | 业界标准的数据可视化库 |
| 后端框架 | FastAPI (Python) | 高性能异步框架，原生支持 SSE |
| 数据库 | MySQL | 存储结构化数据（分类、内容、路径等） |
| 向量数据库 | ChromaDB | 存储文档语义向量，支持相似度检索 |
| 文本提取 | PyMuPDF / python-docx / python-pptx | 支持 PDF、Word、PPT 文本提取 |
| AI 模型 | Ollama 本地部署 | 数据不出服务器，保障隐私安全 |

## 六、典型使用场景

### 场景一：学生自主复习

1. 学生进入教学资料，从目录中选择本周的课程 PDF
2. 阅读过程中遇到不理解的内容，点击 AI 助教按钮
3. 输入问题："第 12 页的公式怎么推导？"
4. AI 基于 PDF 内容给出详细解答，并标注【第12页】
5. 学生点击页码链接，PDF 自动跳转到对应位置对照学习

### 场景二：教师整理教学资源

1. 教师切换到管理模式
2. 创建分类："第三章 — 数据结构"
3. 上传课件 PDF 和教学视频
4. 在知识地图中创建"链表""栈""队列"等知识点节点
5. 建立节点之间的关联关系
6. 创建学习路径："数据结构入门"，按顺序添加步骤
7. 发布后学生即可看到完整的学习方案

### 场景三：知识体系浏览

1. 学生打开知识地图标签页
2. 看到所有知识点以图谱形式展示
3. 点击感兴趣的节点，右侧面板展示该知识点的描述和关联资料
4. 直接跳转到对应学习资料开始学习
